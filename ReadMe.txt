# AESWin - 多通道声发射信号采集与定位系统

AESWin 是一款基于 C++ 和 Microsoft 基础类库 (MFC) 开发的专业声发射 (Acoustic Emission, AE) 监控软件。该系统集成了实时信号采集、参数化硬件控制、波形分析以及高精度的声源定位算法，适用于结构健康监测、无损检测等工业与科研场景。

## 核心功能

* **多通道同步监控**：支持多达 8 个物理通道的并行数据采集与实时波形显示。
* **高精度声源定位**：
    * 支持线性 (1D)、平面 (2D) 及立体 (3D) 定位模式。
    * 可自定义探头坐标结构（如平面或立方体布局）及不同维度的声速参数。
* **硬件参数深度配置**：
    * 各通道独立设置：包括触发阈值、硬件增益、电压量程及通道名称。
    * 数字信号处理：支持软件层的数字增益补偿及高低通滤波上限/下限设置。
* **灵活的采样控制**：支持自定义采样率、单次读取点数及信号处理周期。
* **自动化配置管理**：系统参数通过 `setup.ini` 文件自动保存与加载，确保实验环境的一致性。

## 技术架构

### 软件模型
项目采用成熟的 **MFC 单文档界面 (SDI)** 架构，实现了逻辑业务与数据展示的分离：
* **文档 (AESWinDoc)**：处理核心数据的序列化及 `.lay` 文件的保存与加载。
* **视图 (AETabView)**：基于标签页的多维数据展示界面。
* **主框架 (CMainFrame)**：负责采集流程控制及 UI 消息调度。

### 核心模块
* **采集引擎**：利用 `CRawSignalThread` 进行异步采集，通过 `CRawSignalBuf` 进行高效的数据缓冲管理。
* **驱动接口**：通过 `CArtCardDriver` 模块与底层数据采集卡进行硬件交互。
* **算法逻辑**：`CAELocation2D3D` 模块实现了声波到达时差 (TOA) 定位算法。

## 关键数据结构

### 通道配置 (`CHANNEL_Item`)
定义了通道的物理与逻辑属性，包括：
* `iMonitor_Flag`: 监控开关。
* `iTiggerYuValue`: 触发阈值（单位：dB）。
* `mUperLimit / mLowerLimit`: 数字滤波频率带宽。

### 定位参数 (`AUDIO_LOCATE`)
涵盖了定位计算所需的物理背景信息：
* `iLocateType`: 定位类型（0: 线性, 1: 平面, 2: 立体）。
* `sProbePara`: 存储各通道探头的 X, Y, Z 三维坐标。

## 快速入门

### 开发环境
* **操作系统**: Windows XP 或更高版本。
* **IDE**: Visual Studio (支持 VC++ 项目格式)。
* **库依赖**: MFC (共享 DLL 或静态链接), OLE 支持, 以及可能的第三方信号处理库（如 FFTW）。

### 配置文件
软件在启动时会读取可执行文件目录下 `configure\setup.ini` 中的 `[系统设置]` 节区。您可以直接在该文件中预设采样率、读取点数等全局参数。

### 使用说明
1.  **初始化**：启动程序后，系统会自动初始化采集卡驱动及日志管理器。
2.  **参数设置**：通过菜单进入“通道设置”或“采样参数设置”配置硬件参数。
3.  **开始运行**：点击工具栏的“开始采集”或“开始定位”按钮，系统将开启采集线程并实时处理信号。

## 文件规范
* **项目文件**: `AESWin.vcxproj`。
* **数据文件**: 默认使用 `.lay` 扩展名存储实验布局与数据。
* **资源定义**: `Resource.h` 存储所有界面控件及菜单的 ID 定义。
